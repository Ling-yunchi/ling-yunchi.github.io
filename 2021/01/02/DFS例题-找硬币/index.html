# 找硬币

描述：给定要找零的金额 x, 现有硬币面值为 1,2,4,8,16...(以此类推) 的硬币各两枚。请问一共有多少种找零的方案

# 思路

从大的开始往下分的话会非常繁琐，并且对两块同时分割的话硬币数量的限制会照成很大麻烦இ௰இ

所以我们考虑从下往上推，使用 DFS

# 状态

设总钱数为 M,i 表示要找的钱数，j 表示此时能用的最小硬币面值

f[i][j]f[i][j]f[i][j] 表示用面值最小为 j 的硬币的找 i 元钱时的找零总数

使 j 为能使用的最小硬币面值是为了避免从大往小拆时硬币数量的限制，所以我们进行到f[i][j]f[i][j]f[i][j] 状态时就不必考虑使用比 j 更小的硬币找零的情况了

所以无论前面用了几枚某个面值的硬币，都与接下来的计算无关了 qwqq

# 转移方程

当到达f[i][j]f[i][j]f[i][j] 的状态时，前面的(M−i)(M-i)(M−i) 元钱已经找完，并且不再使用面值比jjj 小的硬币.

那么有三种情况

  1. 不使用面值为 j 的硬币
  2. 使用一枚面值为 j 的硬币
  3. 使用两枚面值为 j 的硬币

f[i][j]=f[i][2∗j]+f[i−j][j∗2]+f[i−2∗j][2∗j]f[i][j]=f[i][2*j]+f[i-j][j*2]+f[i-2*j][2*j]f[i][j]=f[i][2∗j]+f[i−j][j∗2]+f[i−2∗j][2∗j]

# 边界情况

第一次要找的钱数就为 0 时，方法数为 0 特判一下 (¬‿¬)

当不是第一次要找的钱数为 0 时，说明刚好找完，该方法存在。返回 1

当要找的钱数小于硬币面值时无法拼出，说明这种找钱的方法不存在，返回 0

# 转移方程最终

f[i][j]={1i=00i<jf[i][2∗j]+f[i−j][j∗2]+f[i−2∗j][2∗j]i≠0且i≥jf[i][j]=\begin{cases} 1 & i = 0 \\ 0 & i < j \\ f[i][2*j]+f[i-j][j*2]+f[i-2*j][2*j] & i\neq0且i\geq j\\ \end{cases}f[i][j]=⎩⎪⎪⎨⎪⎪⎧​10f[i][2∗j]+f[i−j][j∗2]+f[i−2∗j][2∗j]​i=0i<ji=0且i≥j​

# 题解

#include<iostream>
using namespace std;
int chaifen(int x, int b)
{
	if (x == 0)
		return 1;
	else if (x < b)
		return 0;
	else
	{
		return chaifen(x - b, 2 * b) + chaifen(x - 2 * b, 2 * b) + chaifen(x, 2 * b);
	}
}
int main()
{
	int n;
	cin >> n;
	if (n == 0)
		cout << "0";
	else
	{
		int a = chaifen(n, 1);
		cout << a;
	}
	return 0;
}
