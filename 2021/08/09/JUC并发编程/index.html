# JUC 并发编程

# 什么是 JUC

java.util.concurrent

JUC 就是 java.util .concurrent 工具包的简称。这是一个处理线程的工具包。

目的就是为了更好的支持高并发任务，让开发者利用这个包进行的多线程编程时可以有效的减少竞争条件和死锁线程。

# 进程和线程

进程：一个程序。

一个进程往往可以包含多个线程，至少包含一个！

Java 默认有几个线程？ 2 个 mian、GC

线程：是进程中的实际运作单位。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。

Java 无法自己开启线程

并发、并行

并发：cpu 单核模拟多核执行多线程，在多个线程中来回切换造成同时进行的假象

并行：cpu 多核参与，但每个核只运行一个线程，是真正的线程同时运行

AMD Yes！！！

并发的本质：充分压榨利用 CPU 资源

线程的状态

wait/sleep

  1. 来自不同的类
    • wait => Object
    • sleep => Thread
  2. 锁的释放
    • wait 释放锁
    • sleep 抱着锁睡觉
  3. 使用的范围不同
    • wait 必须在同步代码块或同步控制方法中
    • sleep 可以在任何地方睡

# Lock 锁

传统 synchronized

Lock

公平锁：多个线程按照申请锁的顺序去获得锁，线程会直接进入队列去排队，永远都是队列的第一位才能得到锁。

非公平锁：多个线程去获取锁的时候，会直接去尝试获取，获取不到，再去进入等待队列，如果能获取到，就直接获取到锁。

区别

  • synchronized
    • 内置 Java 关键字
    • 无法获取判断锁的状态
    • 会自动释放锁
    • 线程阻塞时会一直等下去、直到获取锁
    • 可重入锁、不可中断、非公平
    • 适合锁少量的代码同步问题
  • Lock
    • Java 类
    • 可以获取判断锁的状态
    • 必须要手动释放锁、否则会造成死锁问题
    • Lock 不一定会一直等待下去
    • 可重入锁、可判断、可选择公平非公平锁
    • 适合锁大量的同步代码
